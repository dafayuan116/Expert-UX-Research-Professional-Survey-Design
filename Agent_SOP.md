# Agent 核心工作流 (SOP) 与实战示例

## 第一阶段：目标建模 (Goal Specification)

任务：拆解业务目标为心理学/行为学构念。

示例：

- 用户需求：“我想看用户愿不愿意为我们更高级的设计付费。”

- Agent 建模：将其转化为三个构念：[审美愉悦感]（感官认知）、[品牌溢价感知]（价值判断）、[支付意愿阈值]（行为意图）。

## 第二阶段：构念操作化 (Operationalization)

任务：将抽象构念转化为可观察、具体的指标。

示例：

- 抽象构念：[审美愉悦感]

- 操作性定义：不再问“你觉得美吗”，而是转化为：“在观看该界面时，用户感知的视觉平衡程度，以及是否愿意将其作为壁纸或推荐给他人欣赏的程度。”

## 第三阶段：蓝图规划 (Blueprint & ToS)

任务：利用双向细目表规划题目分布，确保内容效度。



示例：

双向细目表矩阵：逻辑 “ 支付意愿更偏向行动，所以增加“意图”层的题目占比。”

## 第四阶段：题库原创与改写 (Item Generation)

任务：生成初稿，包含反向题设计。

示例：

- 正向题：“我愿意为该产品的外观设计支付额外的溢价。”

- 反向题：“如果该产品的功能保持不变，我不会仅因为外观的改变而多花一分钱。”（用于拦截“跟风勾选”的被试）。

## 第五阶段：心理学专家审计 (Professional Audit)

任务：查杀语义 Bug，增强区分度。

示例：

- 原始题目：“这个设计难道不比竞品更高端且易用吗？”（双重负载 + 引导性提问）。

- 审计修正：“相比于您目前使用的产品，该界面的视觉质感给您的感觉是：[1 极低 - 7 极高]。”（去偏 + 连续量表化）。

- 陷阱题植入：“请在本题直接选择‘非常同意’，以证明您正在认真阅读题目。”

## 第六阶段：统计适配交付 (Statistical Handover)

任务：配置背景变量，输出清洗手册。

示例：

- 背景项配置：自动增加“职业领域（设计/非设计）”，以便后期分析【专业审美】与【大众审美】的溢价差异。

- 清洗指南：告知用户：“若受测者在第四阶段的正反两道题中得分一致（如都选 5 分），请作为矛盾样本予以剔除。”

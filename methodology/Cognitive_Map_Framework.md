# Agent 认知地图：
本框架定义了本 Agent 在辅助用户进行问卷设计时的思维导航逻辑，确保设计的每一道题目在“业务价值”与“心理科学”之间保持高度一致。

## 业务情境维度 (Business Context Layer)
Agent 需首先定位用户处于哪种商业/设计节点：

孵化态（探索性）：侧重“美学溢价”、“吸引力”、“心理预期”。

迭代态（诊断性）：侧重“流失风险”、“功能摩擦”、“满意度短板”。

决策态（验证性）：侧重“价格敏感度 (PSM)”、“功能优先级 (Kano)”。

运营态（细分性）：侧重“用户动机”、“生活方式”、“价值观分群”。

## 心理/行为构念维度 (Construct & Behavioral Layer)
Agent 必须将业务目标拆解为可观察的指标：

认知 (Cognitive)：知觉、注意力、偏见、记忆、品牌心智。

情绪 (Affective)：压力、倦怠、审美喜悦、焦虑感、安全感。

意图 (Conative)：购买意向、离职意向、推荐意愿 (NPS)、支付意愿。

行为 (Behavioral)：使用频率、任务完成时长、点击偏好。

## 统计适配维度 (Statistical Compatibility Layer)
Agent 出题时必须预设该题目在报告中的统计角色：

人口统计学（控制变量）：性别、年龄、工龄、部门。

自变量 (Predictors)：如“界面美感评分”、“交互流畅度”。

因变量 (Criterion/Outcome)：如“推荐意愿”、“溢价接受度”。

调节变量 (Moderators)：如“用户专业水平（小白 vs 专家）”。

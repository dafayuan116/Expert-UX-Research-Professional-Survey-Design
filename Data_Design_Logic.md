# 调研设计中的数据类型选择逻辑

在 UX 研究中，题目设计不应是随机的。本项目遵循“决策深度对齐数据类型”的原则，确保每一道题产出的数据都能支撑后续的商业决策。

---

## 1. 数据阶梯：从“描述”到“预测”

我们根据统计能力的强弱，将数据分为四个阶梯。在设计问卷时，我们会根据研究目标的深度来选择相应的“阶梯”。

| 数据级别 | 类型 (Type) | 核心目标 | 统计能力 (Power) | 典型应用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **Level 1** | **定类 (Nominal)** | **分类** | 频数、百分比、众数 | 用户画像（性别、职业）、A/B方案偏好强选 |
| **Level 2** | **定序 (Ordinal)** | **排序** | 中位数、等级相关 | 需求优先级排序、功能重要度对比 |
| **Level 3** | **定距 (Interval)** | **程度** | **均值、标准差、相关性分析** | 满意度量表（李克特5/7点）、易用性认知 |
| **Level 4** | **定比 (Ratio)** | **数值** | **倍数、弹性分析、回归预测** | 愿意支付的价格（PSM）、使用时长、点击次数 |

---

## 2. 场景化题型配比规律 (Scenario Rules)

根据业务周期的不同阶段，本项目 Agent 会自动应用以下配比逻辑：

### A. 新品孵化（侧重吸引力与溢价）
* **策略**：**定距 + 定比**
* **逻辑**：用定距量表测量“美学吸引力”的绝对值，用定比数据（直接填空）测算“商业溢价”，以支持 PSM 价格模型。

### B. 产品迭代（侧重流失与痛点）
* **策略**：**定距 + 定序**
* **逻辑**：通过全旅程定距量表扫描找出“低分环节”（痛点），随后通过定序题目让用户对痛点进行“体感排序”，确定研发优先级。

### C. 商业决策（侧重转化与定价）
* **策略**：**定比核心化**
* **逻辑**：严禁在核心定价变量上使用“区间单选”（定类），必须强制使用“数值填空”（定比），以确保可以进行高阶回归分析。

### D. 用户运营（侧重画像与动机）
* **策略**：**定类最大化**
* **逻辑**：通过多维度的定类题目（生活方式、价值观标签）进行聚类分析（Cluster Analysis），实现精准的用户分群。

  
## 广域研究场景与数据类型映射表 

| 研究取向 | 具体场景 | 核心数据类型策略 | 统计学价值 |
| :--- | :--- | :--- | :--- |
| **1. 探索/认知** | 心智模型表征 | 定类 + 开放文本 | 建立编码体系，识别用户分类与心智锚点。 |
| **1. 探索/认知** | 认知负荷与效能 | 定距 (7点/9点) | 适配 NASA-TLX 逻辑，计算加权负荷总分。 |
| **2. 情感/体验** | 情感依恋与共鸣 | 定距 (李克特量表) | 进行相关分析，验证情感变量对预测力。 |
| **2. 情感/体验** | 审美与知觉经验 | 定距 + 定比 (滑块) | 测算审美偏好的绝对强度或视觉阈限。 |
| **3. 动机/社会** | 行为背后的动机 | 定距 (SDT 专用量表) | 计算内生/外生动机指数，支持路径分析。 |
| **3. 动机/社会** | 社会态度与偏见 | 定类 + 定距 (强迫选择) | 识别群体差异与社会称许性偏误。 |
| **4. 评价/干预** | 系统效能审计 | 定距 (SUS/UMUX 标准项) | 计算系统可用性百分位分数，对齐行业基准。 |
| **4. 评价/干预** | 干预效果评估 | 定距 (配对设计) | 比较“前测-后测”基线变化，验证干预有效性。 |

## 3. 判别准则：我该选哪种数据？

在添加新题目时，本项目遵循以下决策树：

1.  **我是否需要计算平均分？**
    * Yes $\rightarrow$ 必须使用 **定距（Interval）**。
2.  **我是否需要做不同人群的评价对比？**
    * Yes $\rightarrow$ 必须配置 **定类（Nominal）** 画像题作为自变量。
3.  **我是否需要预测具体的商业回报（如收入增长）？**
    * Yes $\rightarrow$ 核心变量必须使用 **定比（Ratio）**。

# 用户研究方法 - 问卷调研

在当今的商业决策中，问卷调研是最常用的工具，但也常因为设计的非专业性导致 “垃圾进，垃圾出 (GIGO)”。许多问卷在设计阶段就由于变量水平设置错误、题目逻辑混乱或样本无法拉开差距，导致后期无法进行相关分析、回归分析等高阶统计。

本项目基于心理测量学 (Psychometrics) 的专业理论，讲量表编制中项目分析 (Item Analysis) 的逻辑引入问卷设计流程，旨在确保每一份问卷在分发前就具备了支撑科学决策的统计学精度。

“示例” 

普通 AI 出题：“你觉得我们的设计好吗？”（诱导性，无效）

本 Agent 出题：“在完成任务 A 时，该界面的视觉布局对您的干扰程度是：...”（操作化，专业）

## 使用场景

新品孵化阶段：需要测算设计美学的商业溢价与用户吸引力。

产品迭代阶段：需要通过全旅程满意度调研定位流失风险点。

商业决策阶段：需要基于 PSM 模型测算最优定价区间。

用户运营阶段：需要基于心理动机和行为频率进行用户聚类分群。

## 目标用户

UX设计师/产品经理：希望通过数据证明设计价值，但缺乏专业用研背景。

初级研究员：需要一个专家级的审计工具来确保题目逻辑自洽、信效度达标。

AI+UX 实践者：探索如何利用 AI 提升用户洞察的严谨性。


## 使用方法

### 1. 业务人员：直接“复制粘贴”运行

这是门槛最低的使用方式。如果你是（如 PM 或运营）想快速出一份专业问卷：

操作： 打开任意主流 AI（如 Claude、ChatGPT、Gemini）。

指令： 先把 Prompts.md 里的所有内容粘贴进去发送给 AI，然后再输入一句业务需求（例如：“我想测测用户对新版图标的喜爱程度”）。

效果： AI 会立刻变身为“心理测量专家”，按照你六步 SOP 吐出完整的量表草案。

### 2. 开发者：作为“后端逻辑”集成

如果想把这 Agent 做到公司内部的某个调研平台上：

操作： 

- 1.  开发者在代码中读取 Prompts.md 的文本。
     
- 2.  将其作为 system_prompt 传递给 LLM 接口。
     
- 3.  因为你在 Prompts.md 里规定了 JSON 输出格式，程序可以直接解析 AI 返回的结果，自动生成问卷页面或存入数据库。

价值： 你的文档就是他们的“产品需求文档 (PRD)”和“API 指南”。

### 3. 研究者：作为“方法论”参考

如想了解设计逻辑：

操作： 查阅 methodology/ 文件夹下的文档。

价值： 可以了解如何将“认知地图”和“心理测量学”结合的。

  
## 交付物 (Deliverables)

完成设计后，您将获得一个压缩包式的输出，包含：

Markdown 格式问卷本：可直接复制到问卷星/腾讯问卷。

SPSS/Python 预处理逻辑：标注哪些题目需要反向计分（Reverse Scoring）。

数据清洗判据：明确陷阱题的剔除标准。

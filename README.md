# 用户研究方法 - 问卷调研

在当今的商业决策中，问卷调研是最常用的工具，但也常因为设计的非专业性导致 “垃圾进，垃圾出 (GIGO)”。许多问卷在设计阶段就由于变量水平设置错误、题目逻辑混乱或样本无法拉开差距，导致后期无法进行相关分析、回归分析等高阶统计。我基于心理测量学 (Psychometrics) 的专业理论，讲量表编制中项目分析 (Item Analysis) 的逻辑引入问卷设计流程，旨在确保每一份问卷在分发前就具备了支撑科学决策的统计学精度。
本仓库提供了一套基于 Multi-Agent (多智能体) 架构 的专业量表编制与审计解决方案。系统集成了心理测量学（Psychometrics）严谨性与 UX 研究的灵活性，旨在将模糊的业务意图转化为具备“证据级”效力的量表工具。

## 使用场景

### 1. 探索性/认知研究 (Exploratory & Cognitive)
- **心智模型表征**：研究用户对复杂系统的认知映射，关注理解偏差与心理期望。
- **认知负荷与效能**：测算特定任务下的心理资源占用（基于 NASA-TLX 逻辑），研究人机交互的疲劳度。

### 2. 情感与体验心理 (Affective & Experience)
- **情感依恋与共鸣**：研究个体与客体（产品/他人/环境）之间的情感纽带、归属感及社会存在感。
- **审美与知觉经验**：侧重视觉感知的绝对阈限、审美喜悦及其对个体心理状态的补偿作用。

### 3. 行为动机与社会心理 (Motivation & Social)
- **行为背后的动机**：基于 自决理论 (SDT)，深挖行为驱动力（内生 vs 外生）与价值取向。
- **社会态度与偏见**：研究群体间的内隐态度、社会称许性影响及价值观分群。

### 4. 评价与干预研究 (Evaluation & Intervention)
- **系统效能审计**：基于标准量表（SUS/UMUX）对特定交互系统进行可用性压力测试。
- **干预效果评估**：针对某一改变（设计/政策/教育）前后的心理变量基线对比。

## 目标用户

UX设计师/产品经理：希望通过数据证明设计价值，但缺乏专业用研背景。

初级研究员：需要一个专家级的审计工具来确保题目逻辑自洽、信效度达标。

AI+UX 实践者：探索如何利用 AI 提升用户洞察的严谨性。


## 系统架构 

系统采用**“建模(A) -> 编制(B) -> 审计(C)”**的串联流水线逻辑，确保调研设计在发布前经过三重专业复核：

| 专家角色 | 阶段 | 核心任务 | 交付物 |
| :--- | :--- | :--- | :--- |
| **Agent A: Architect** | **建模层** | 需求场景对齐、构念操作化 | 研究建模委托单 |
| **Agent B: Builder** | **执行层** | 题库生成、反向题/陷阱题埋点 | 问卷初稿 & 清洗判据 |
| **Agent C: Auditor** | **质检层** | 基于 Logic Matrix 的五维压力测试 | 审计反馈 & 最终版本 |

## 使用指南

### 模式一：端到端量表创建 (从 0 到 1)

启动：输入您的研究意图（如：“我想调研新版 AI 功能的使用动机”）。

场景对齐：Agent A 会引导您从四大业务场景（新品孵化、产品迭代、商业决策、用户运营）中选一。

确认建模：确认 Agent A 输出的建模委托单。

生成与审计：Agent B 会自动出题，随后 Agent C 会执行质检并输出最终可直接发布的问卷。

### 模式二：存量问卷专家审计 (专项质检)

启动：直接输入您已写好的题目或问卷草稿。

审计：系统会自动跳过建模阶段，由 Agent C 直接输出 [原题扫描 -> 逻辑冲突 -> 改版方案 -> 价值背书]。
  
## 交付物 (Deliverables)

完成设计后，您将获得一个压缩包式的输出，包含：

Markdown 格式问卷本：可直接复制到问卷星/腾讯问卷。

SPSS/Python 预处理逻辑：标注哪些题目需要反向计分（Reverse Scoring）。

数据清洗判据：明确陷阱题的剔除标准。
